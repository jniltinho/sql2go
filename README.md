# sql2go

A command-line tool written in Go to automatically generate model code and `structs` from MySQL database tables.

Under the hood, this project uses the official GORM generator, [`gorm.io/gen`](https://gorm.io/gen/gen_tool.html).

## Overview

`sql2go` uses Cobra CLI, connects to a MySQL database, generates Go models for all tables, removes the intermediate `query` package, and optionally merges all generated models into a single file.

## Requirements

- Go installed locally
- Access to a MySQL database
- [UPX](https://upx.github.io/) installed if you want to use the compressed binary generated by the `Makefile`

## Build

Build the project with:

```bash
make build
```

This creates the binary at `bin/sql2go`.

The binary embeds:

- `Version`: resolved from the latest git tag, or `dev`
- `BuildDate`: generated at build time in UTC

If you do not want to use `make`, you can build it directly with Go:

```bash
go build -o bin/sql2go main.go
```

## Usage

### Show CLI help

```bash
./bin/sql2go --help
```

### Show version information

```bash
./bin/sql2go version
```

### Run with Makefile

Run with the default Makefile variables:

```bash
make run
```

Run with custom values:

```bash
make run DB_NAME=my_database USER=root PASS=my_password OUT_DIR=./models
```

Run with merge enabled:

```bash
make run DB_NAME=my_database USER=root PASS=my_password OUT_DIR=./models MERGE=true
```

### Run the binary directly

```bash
./bin/sql2go generate -db my_database -host 127.0.0.1 -port 3306 -user root -pass "my_password" -out ./models
```

Enable file merge explicitly:

```bash
./bin/sql2go generate -db my_database -host 127.0.0.1 -port 3306 -user root -pass "my_password" -out ./models -merge
```

## CLI Flags

- `-db`: Database name, required
- `-host`: MySQL host, default `127.0.0.1`
- `-port`: MySQL port, default `3306`
- `-user`: MySQL user, default `root`
- `-pass`: MySQL password, default empty
- `-out`: Output directory for generated models, default `./models`
- `-merge`: Merge generated models into a single file, default `false`

## Commands

- `sql2go`: Show help
- `sql2go generate`: Generate models from the target MySQL database
- `sql2go version`: Print the embedded version and build date
- `sql2go --help`: Show the available commands and flags

## How It Works

1. Connects to the target MySQL database.
2. Generates models for all tables using `gorm.io/gen`.
3. Writes the generated files to the output directory.
4. Removes the temporary `query` folder.
5. Optionally merges all generated model files into a single `<database>.go` file when `-merge` is enabled.
